# 15 — n8n Workflow Automation

[n8n](https://n8n.io) — open-source workflow automation platform. Connects services via visual flows (webhooks, schedules, API calls, etc.).

## Stack

| Container | Image | Purpose |
|-----------|-------|---------|
| `n8n` | `n8nio/n8n:latest` | Automation engine + web UI |

Single container, SQLite database (no separate DB needed at homelab scale).

## Install

[`compose/n8n.yml`](../compose/n8n.yml).

```bash
mkdir -p /mnt/drive1/AppData/n8n
chown -R 1000:1000 /mnt/drive1/AppData/n8n
```

The `chown` is required — n8n runs as UID 1000 inside the container and needs write access on first boot to generate its encryption key.

```bash
docker compose -f /mnt/drive1/AppData/compose/n8n/docker-compose.yml up -d
```

## Configuration

Edit the env vars in the compose file before starting:

| Variable | What to set |
|----------|-------------|
| `N8N_HOST` | Your subdomain, e.g. `n8n.yourdomain.com` |
| `WEBHOOK_URL` | Full URL with trailing slash, e.g. `https://n8n.yourdomain.com/` |
| `GENERIC_TIMEZONE` | Your TZ, e.g. `America/New_York` |

## Data location

```
/mnt/drive1/AppData/n8n/
├── config           # encryption key (auto-generated on first boot)
├── database.sqlite  # all workflows, credentials, executions
├── nodes/           # community nodes
└── storage/         # binary data from executions
```

## Nginx Proxy Manager

Add a Proxy Host:

- **Domain:** `n8n.yourdomain.com`
- **Forward Host/IP:** `192.168.50.178` (LAN IP — not container name, different Docker network)
- **Forward Port:** `5678`
- **Websockets Support:** enabled (required for the n8n editor)
- SSL: enable and request cert if you have a wildcard or Let's Encrypt setup

## Notes

- The "Python task runner" warning on startup is harmless — Python isn't installed in the image, but JavaScript/Node runners still work.
- `N8N_SECURE_COOKIE=false` is set so the HTTP connection from NPM to n8n doesn't cause cookie auth failures.
- Backups: `database.sqlite` is the only critical file — include `/mnt/drive1/AppData/n8n/` in your backup script.
