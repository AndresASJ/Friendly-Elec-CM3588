# 2026-06-02

## Todo Capture: auto-file captures into Todoist sections

Enhanced the **Todo Capture → Todoist** n8n flow so captured tasks land in the right
**section** instead of piling up loose in Inbox-root. Previously Gemini's category was
only applied as a *label*; the task itself always went to Inbox with no section, so
everything texted to the ASJ Noti bot accumulated unsorted (this is what prompted the
big Todoist cleanup below).

### Change

Two edits to [`configs/n8n-workflow-todo-capture-todoist.example.json`](../configs/n8n-workflow-todo-capture-todoist.example.json):

- New **Get sections** node (`GET /api/v1/sections`, reuses the Todoist API
  credential) inserted between **Lookup contact** and **Build task**.
- **Build task** now matches Gemini's category (case-insensitive) against the fetched
  section names and sets `section_id` when one matches (Blog / Church / Coding / Kord /
  Routines). Updated the Gemini prompt in **Get message** to bias categories toward
  those real section names.

Design notes:
- **No hardcoded section IDs** — resolved at runtime, consistent with the flow's
  existing "no hardcoded project IDs" principle. Renaming/reordering sections won't
  break it.
- **Label only when unmatched** — if a category maps to a section, the task is *not*
  also tagged with a same-named label (the section/label redundancy is exactly what we
  just cleaned). Unmatched categories still fall back to a label.

Apply step (no n8n public API key configured): re-import the example in n8n → Workflows
→ Import from File, re-attach the Telegram / Gemini / Todoist / Postgres credentials
(the `CHANGE_ME` placeholders), and delete the old copy. Doc updated in
[`docs/15-n8n.md`](../docs/15-n8n.md).

## n8n backlog: Yelp Auto-Reply on hold

Researched the Yelp Auto-Reply idea. Yelp's **Leads API** is the only one that can post
replies (`POST /v3/leads/{lead_id}/events`), it's **partner-gated** (OAuth + program
agreement), and new-lead discovery is **webhook-only** (no polling) — which the
LAN-only box can't receive without a tunnel. Parked until the owner secures Leads API
access; build plan captured in memory. Several backlog items were culled the same day;
active focus is the ASJ Blog flows.
