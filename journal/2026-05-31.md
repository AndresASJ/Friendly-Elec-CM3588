# 2026-05-31 — n8n: Todoist capture bot + Things migration

## Summary

Built a two-way **Todoist capture bot** (text the ASJ Noti bot → Gemini cleans it
→ Todoist task) and a one-time **Things 3 → Todoist importer**. Also hardened the
capture flow after it dropped messages in real use.

## Changes

- **Todo Capture → Todoist (via Gemini)** (workflow `r5gc1hgtCyimoN5v`):
  Telegram polling → `gemini-3.5-flash` extracts {title, due, label, priority}
  → Todoist `POST /api/v1/tasks` → ✅ reply.
  - Uses **outbound `getUpdates` polling** (not a Telegram Trigger) because
    n8n.asj.media is LAN-only and Telegram can't reach an inbound webhook.
  - Added **Gemini API key** (HTTP Header `x-goog-api-key`) and **Todoist API**
    (Bearer) credentials.
- **Things → Todoist Import** (workflow `EsEMth8aREpDrYcn`): webhook receives an
  Apple-Shortcut payload of open to-dos and creates them in Todoist, deduped,
  with Things Area/Project preserved as `Things_*` labels.
  - Migrated **40 tasks**; later organized into Todoist **Sections** (Kord, Blog,
    Church, Coding, Routines) and deleted the 14 guitar/theory "music" tasks.
- Sanitized exports: `configs/n8n-workflow-todo-capture-todoist.example.json`,
  `configs/n8n-workflow-things-import.example.json`.

## Why

Wanted to capture todos by texting the existing bot, and to move off Things 3
(Apple-only, no API) onto Todoist (full REST API, iOS widgets).

## Gotchas (all hit live, all fixed)

- **Gemini model:** the consumer Gemini *subscription* has no API. Must use an
  **API key** from AI Studio (free tier). Model is `gemini-3.5-flash` (newer than
  my assumptions — verified against the live `/models` list).
- **n8n expression `}}` collision:** building the Gemini JSON body inline as
  `={{ JSON.stringify({...}) }}` breaks because nested `}}` ends the expression.
  Fix: build the body string in a Code node, reference it as `={{ $json.geminiBody }}`.
- **Email→Things dead end:** "Mail to Things" only supports title+notes — no
  dates/projects/tags. Pivoted to Todoist for structured fields.
- **Proton SMTP:** only works on paid plans via SMTP submission tokens
  (`smtp.protonmail.ch:587`); free Proton has no SMTP. (Used briefly for the
  Things email experiment, since dropped in favor of Todoist.)
- **Lost messages (6 sent, 1 captured):** the offset was advanced when messages
  were *read*, before tasks were created — so a Gemini rate-limit (429) on a burst
  silently dropped 5. **Fix:** process **one message per poll** and advance the
  offset only in a final **Mark read** node *after* the Todoist task is created.
  Now nothing is lost and nothing is duplicated.
- **404 Project not found:** capture had hardcoded Home/Education project IDs;
  those Todoist projects were later deleted. **Fix:** capture into Inbox and apply
  the category as a **label** (auto-creates, never 404s).
- **Execution history is useless for a 1-min poller** — successful runs are pruned
  (last 20 kept), so message-processing runs roll off within ~20 min. Debugged via
  the Telegram offset + Todoist state instead.

# 2026-05-31 — FlacPlayer TODO.md → Todoist mirror

## Summary

Mirrored the open tasks from the private `AndresASJ/FlacPlayer` repo's `TODO.md`
into Todoist (label `FlacPlayer`), one-way, file as source of truth. 77 tasks
imported; hourly cron keeps it current.

## Changes

- `scripts/flacplayer-todo-sync.py` → `/usr/local/bin/` — fetches `TODO.md` via
  `gh` (reuses the box's GitHub auth, no new token), parses open `- [ ]` items
  (title = first sentence, rest → description, tier prefixed), creates missing
  Todoist tasks and completes ones no longer open in the file.
- Todoist token stored at `/root/.config/flac-sync/todoist.token` (chmod 600).
- Cron (root): `7 * * * *` (hourly).
- 77 tasks created under the `FlacPlayer` label; sync verified idempotent (0/0
  on re-run).

## Why

User wanted their FlacPlayer dev TODO synced with Todoist. TODO.md is a 600-line
tiered design doc — two-way sync would mangle it, so one-way mirror (file = truth)
is the safe, useful choice.

## Gotchas

- **Todoist strips `@word`/`#word` from task `content`** (parses them as label /
  project syntax) — e.g. "Drop unbounded @Query fetches" stored as "Drop unbounded
  fetches" + a junk `Query` label. Caused endless create/complete churn until
  caught. Fix: **backslash-escape** `@`/`#` in titles (zero-width space did NOT
  work; Todoist still parsed it).
- Matching by title text is therefore unreliable — switched to a stable
  **`fpid:<md5>` marker in the description** (descriptions aren't parsed). Now
  idempotent.
- Todoist `GET /tasks` **paginates** (50/page via `next_cursor`); the first
  delete pass missed ~27 tasks until pagination was handled.

## Files touched

- `scripts/flacplayer-todo-sync.py` (new)
- `docs/15-n8n.md` (FlacPlayer mirror section)
- `journal/2026-05-31.md` (this entry)
