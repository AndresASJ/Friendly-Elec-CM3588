# 2026-05-28 — Repo rewrite + journal kickoff

## Summary

Threw out the old single-file `README.md` (which still described a CM3588 with PhotoPrism + Samba + Portainer — none of which we run anymore) and rebuilt the repo from the ground up to mirror the current 30-container stack.

Also stood up a workflow: every service change going forward gets reflected in the repo and a journal entry like this one.

## Changes

- **Wrote 14 step-by-step docs** under `docs/` covering hardware, storage, CasaOS, networking, the *arr stack, VPN-routed downloads, Immich, music, Home Assistant, Seafile, Ghost, Recyclarr, backups, and troubleshooting
- **Added 22 sanitized docker-compose files** under `compose/` — one per service, pulled from the actual running configs with secrets replaced by `CHANGE_ME_*` placeholders
- **Added example configs** for Recyclarr, WireGuard, and the qBit port-forward script
- **Added scripts** for initial host setup, drive UUID discovery / fstab generation, and a nightly backup
- **Added `.gitignore`** that blocks `*.env`, `wg0.conf`, real `recyclarr.yml`, Cloudflare cert JSONs, and DB dumps

## Why

The repo was stale by ~6 months of homelab evolution. Goal was twofold:
1. **Reproducibility** — anyone reading the repo (including future-me) can rebuild the exact stack from scratch
2. **Source of truth** — when adding a new service, the repo is now the place that records "what we run and why"

## Gotchas

- Old README referenced **CM3588**; actual board is **CM5388**. Fixed throughout.
- Discovered three parallel app-data directories with different casings (`/DATA/AppData/`, `/mnt/drive1/appdata/`, `/mnt/drive1/AppData/`). Documented in `docs/02-storage.md` to avoid future confusion.
- First push attempt failed with 403 — initial PAT was created with "Public repositories" (read-only) access. Fixed by switching to "Only select repositories" with Contents: Read and write.
- The recyclarr `.yml` on disk has real Sonarr/Radarr API keys — kept it out of the repo via `.gitignore`, shipped a sanitized `.example`.
