# Journal

Daily logs of homelab changes — what was done, why, and any gotchas.

One file per day: `YYYY-MM-DD.md`. New entries appear in the list below as they're added.

## Format

Each entry follows this skeleton:

```markdown
# YYYY-MM-DD — short title

## Summary
1–2 sentences on the day's work.

## Changes
- Service added / removed / reconfigured (with file paths)

## Why
Context — what problem was being solved, what decision was made.

## Gotchas
Anything that broke, surprised, or needed a workaround.

## Files touched
- compose/foo.yml
- docs/05-media-stack.md
```
